# USER STORIES — Free Sewaa

This document defines the core user stories for the Free Sewaa platform.  
Each story represents a specific user need and is used to guide development, task creation, and sprint planning.

Stories are written in a structured format and include priority, ownership, and clear acceptance criteria to ensure they are actionable and testable.

---

## Story 1 — Homepage Experience (Priority: High)

**User Story**  
As a new user, I want to view a clear and informative homepage, so that I can quickly understand what Free Sewaa offers.

**Acceptance Criteria**
- Homepage loads without errors
- Platform name and purpose are clearly visible
- Navigation to login page is available
- Layout is clean, readable, and responsive

**Owner:** Mohan Khadka  
**Status:** Done  

---

## Story 2 — Email Authentication (Priority: High)

**User Story**  
As a user, I want to log in using my email and password, so that I can securely access the platform.

**Acceptance Criteria**
- User can enter email/username and password
- Password visibility toggle is available
- Validation errors appear for empty or invalid input
- Login button triggers validation and feedback
- Success message is shown after login attempt

**Owner:** Sujan Shrestha  
**Status:** In Progress  

---

## Story 3 — Phone Authentication with OTP (Priority: High)

**User Story**  
As a user, I want to log in using my phone number and OTP verification, so that I can access my account without using a password.

**Acceptance Criteria**
- User can enter phone number with country code
- OTP is generated (frontend simulation)
- User can enter a 6-digit OTP
- Countdown timer is visible
- Resend OTP option is available after timer
- Successful verification logs the user in

**Owner:** Mohan Khadka  
**Status:** In Progress  

---

## Story 4 — Navigation Flow (Priority: Medium)

**User Story**  
As a user, I want to navigate easily between pages, so that I can move through the application without confusion.

**Acceptance Criteria**
- Navigation links are clearly visible
- Homepage and login pages are connected
- Clicking links loads correct pages without errors

**Owner:** Sujan Tamang  
**Status:** Done  

---

## Story 5 — User Interface & Responsiveness (Priority: Medium)

**User Story**  
As a user, I want a clean and responsive interface, so that I can use the platform on both mobile and desktop devices.

**Acceptance Criteria**
- Layout adjusts properly on different screen sizes
- Inputs, buttons, and text are clearly visible
- UI follows a simple, modern design style
- No overlapping or broken elements

**Owner:** Swarnim Jung Karki  
**Status:** In Progress  

---

## Story 6 — Theme Preference (Dark Mode) (Priority: Low)

**User Story**  
As a user, I want to switch between light and dark mode, so that I can use the application comfortably in different lighting environments.

**Acceptance Criteria**
- Toggle button is visible
- Theme changes when toggled
- UI remains readable in both modes

**Owner:** Ram Pathak  
**Status:** Planned  

---

## Story Mapping to Development

Each user story is:

- Converted into GitHub Issues
- Assigned to a team member
- Tracked on the project board (To Do → In Progress → Done)
- Linked with commits and pull requests

---

## Development Approach

- Focus on **small, testable features**
- Prioritize **working functionality over complexity**
- Build incrementally across sprints
- Ensure every story is **demoable**

---

## Summary

These user stories define the foundation of the Free Sewaa platform and guide the team’s development process.  
They ensure that every feature built is aligned with user needs and contributes to a functional, testable system.
