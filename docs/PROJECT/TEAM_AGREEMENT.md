# Team Roles (Rotate Weekly)

Roles will rotate every sprint so that each team member experiences different responsibilities.

### PM (Project Manager)
Responsible for:
- Creating the Weekly Sprint Packet Issue
- Coordinating team tasks
- Ensuring work is visible in GitHub
- Submitting the weekly Sprint Packet

### Scribe
Responsible for:
- Recording team decisions
- Updating documentation
- Tracking blockers and risks
- Maintaining sprint notes

### QA Lead
Responsible for:
- Verifying that implemented features work correctly
- Testing the system before demos
- Ensuring project quality

### Demo Driver
Responsible for:
- Preparing the demo
- Writing the demo script
- Running the live demo if needed

---

# GitHub Workflow Rules

Our team will follow the workflow:

Issue → Branch → Pull Request → Merge → Evidence

Rules:

- No work should begin without a GitHub Issue
- Code should be developed in separate branches
- Changes should be merged through Pull Requests
- Important work should be reviewed by a teammate before merging

---

# Evidence Rule

This course uses an evidence-based workflow.

**Rule:**  
If it isn’t linked, it didn’t happen.

Each team member must provide **2–3 pieces of evidence weekly**.

Examples of acceptable evidence:

- Pull Requests
- Closed Issues
- Code Commits
- Review Comments
- Documentation Updates
- Screenshots of working features

Evidence will be posted as comments on the **Weekly Sprint Packet Issue**.

# Work Expectations

Each team member agrees to:

- Contribute work every week
- Provide visible evidence of their contributions
- Communicate blockers early
- Respect teammates’ time and responsibilities

---

# Missed Work

If a team member misses assigned work:

- They must inform the team as soon as possible
- They must complete the work later
- The missed grade may remain zero depending on course policy

Repeated missed work may lead to instructor intervention.

---

# Conflict Resolution

If a conflict occurs, the team will follow these steps:

**Step 1 – Team Discussion**  
Attempt to resolve the issue respectfully within the team.

**Step 2 – Adjust Responsibilities**  
Tasks may be redistributed if necessary.

**Step 3 – Document the Issue**  
Record the issue in GitHub (Issue or comment).

**Step 4 – Instructor Support**  
If the problem continues, contact the instructor.

---

# Contribution Rule

Each team member must show weekly contributions through:

- Git commits
- GitHub Issues
- Pull Requests
- Documentation updates

All contributions must be visible in the repository.
