# Sprint Notes

## What Shipped

During Sprint 1, the team successfully completed the onboarding phase of the Free Sewaa project.

Completed work includes:
- Creating the team repository
- Opening the project idea pitch issue
- Starting the team agreement
- Creating the Sprint Packet Issue
- Building an initial homepage interface using HTML and CSS

This sprint established the foundation for future feature development and team collaboration.

## What Broke

During repository setup, the team faced a synchronization conflict while pushing files to GitHub because the remote repository already contained existing files.

## How It Was Fixed

The issue was resolved by pulling the latest remote changes, syncing the local repository, and then pushing the updated files successfully.

## Next Sprint Plan

During Sprint 2 (Reset Week), the team will focus on:

- Organizing the GitHub project board
- Setting up issue templates
- Finalizing frontend and backend development environments
- Beginning development of the item listing interface

## Risks / Blockers

Possible challenges include:
- Some team members may need help configuring development environments
- Scheduling team meetings may be difficult
- Backend technology decisions may take additional discussion

The team will manage these risks through communication and collaborative troubleshooting.
