# 🧠 Sprint Notes

## 🚀 What Shipped

During Sprint 5, the project moved from a **basic template to a near real-world product UI**.

Key achievements:

* Complete **frontend redesign** to premium-level UI (Airbnb/Karrot inspired)
* Implemented **dashboard-style homepage after login**
* Added **logout system inside profile/settings**
* Improved **color gradients and visual hierarchy**
* Built **responsive design across all devices**
* Enhanced **button design and animations**
* Added **image upload functionality in donation flow**
* Improved **overall UX flow and navigation consistency**

---

## ❌ What Broke

* UI inconsistencies across pages (different styles)
* Navigation bugs (incorrect redirects)
* Layout issues on mobile devices
* Gradient and color mismatch problems

---

## 🔧 How It Was Fixed

* Standardized **global CSS design system**
* Refactored layout using **Flexbox/Grid**
* Fixed routing/navigation logic in JavaScript
* Unified gradients and color palette across all pages
* Improved responsive breakpoints

---

## 🔜 Next Sprint Plan (Sprint 6)

* Integrate **real backend (Node.js / Firebase)**
* Implement **real-time chat system**
* Add **user profile dashboard (My Items / My Requests)**
* Add **notifications system**
* Improve **security (auth handling, validation)**

---

## ⚠️ Risks / Blockers

* Backend integration complexity
* Real-time chat implementation challenges
* Time constraints for polishing all features
* Team coordination for merging large changes
